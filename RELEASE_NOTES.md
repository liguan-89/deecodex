# DEX AI 版本信息汇总

更新时间：2026-06-23

说明：

- 本文件合并正式版、Preview 版、Beta 版和早期历史版本信息。
- 版本来源以 `git tag`、`release:` 提交和当前发版记录为准。
- `3.3.8-preview.11` 到 `3.3.8-preview.28` 以技术预览 worktree 的 `VERSION`、`Cargo.toml` 和 Tauri 版本文件为准。
- 部分中间版本没有独立 tag，但有明确 release commit，下面按阶段合并说明。
- 内部兼容名仍保留 `deecodex`、`~/.deecodex`、`provider id = deecodex` 等历史字段。

## 当前推荐版本

### 3.6.1

- 新增 Codex 配置智能守护，运行时检测 `config.toml` 被外部工具覆盖后恢复 DEX 管理字段。
- 守护同时支持 API 模式和智能路由模式：API 模式恢复 `deecodex + /v1`，智能路由模式恢复 `dex_router + /codex-router/v1`。
- 守护采用文件事件监听和 debounce，不做每秒轮询，也不锁死整个 Codex 配置文件。
- 设置页新增“智能守护 Codex 配置”开关，便于用户显式控制。

### 3.6.0

- 修复 Codex 模型目录同步失败时会移除 `model_catalog_json`，导致账号模型直选列表消失的问题。
- 模型目录本轮未生成账号模型且账号库不可用时，会保留已有 `dexacct` 直选项，避免被空目录覆盖。
- 停止服务或关闭自动注入不再物理删除 `models_deecodex.json`，降低 API 模式和智能路由切换后列表丢失风险。
- 账号模型目录生成改为 checked 读取账号库，账号文件异常会显式中断同步并复用已有目录。

### 3.5.7

- 修复 `models_deecodex.json` 精简账号模型时删除 Codex 必填字段，导致 Codex Desktop 配置整体加载失败的问题。
- 账号模型目录保留轻量 `base_instructions` / `model_messages` schema，避免再次出现模型列表只剩基础模型、会话索引不生效的情况。
- 保持账号模型目录体积受控，不再复制大段 Codex 原始提示词到每个账号模型条目。

### 3.4.2

- 修复 3.4.1 正式版登录态迁移只触发一次的问题。
- 正式版会按 `~/.deecodex-preview/accounts.json` 文件指纹增量同步，Preview 账号文件变化后会再次迁移。
- 保留 Preview 账号原始 ID，避免 Codex 模型目录里的 `dexacct` 模型 slug 指向正式版不存在的账号。
- Codex 桌面版和 DEX 助手的活跃账号选择会跟随 Preview 当前选择恢复。
- 迁移仍不覆盖正式版全局活跃账号，避免影响 CLI 或旧正式区主账号。

### 3.4.1

- 修复正式版从 Preview 转正后无法继承预览版账号登录态的问题。
- 正式版首次启动时会从 `~/.deecodex-preview/accounts.json` 合并缺失账号到 `~/.deecodex/accounts.json`。
- OAuth 登录态、Codex 桌面版账号选择和 DEX 助手账号选择会随账号迁移一起恢复。
- 迁移不会覆盖正式版已有账号和正式版全局活跃账号。
- 迁移完成后会写入 `preview_accounts_migrated.json` 标记，避免后续反复用 Preview 旧数据覆盖正式版。

### 3.4.0

- 技术预览功能回归主线，发布为正式版。
- Tauri 打包身份从 `DEX AI Preview` 切回 `DEX AI`。
- 正式版使用 `com.deecodex.app`、`deecodex-gui` 和默认端口 `4446`。
- 保留 Preview 版隔离逻辑：Preview 继续使用独立数据目录和端口，便于并行测试。
- 合入到 `3.3.8-preview.28` 为止的 Codex Router、账号模型直选、Computer Use 稳定性和 MiMo/MiniMax 工具链修复。

## 后续维护建议

- 正式版继续从 `3.4.x` 往后发补丁版本。
- 大功能先走 `preview` 或 `beta`，稳定后再回归正式主线。
- 每次发版后同步更新本文件，至少记录版本号、核心改动、兼容性影响和是否需要迁移数据。
