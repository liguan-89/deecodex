## 3.9.9

- mimo Responses 兼容：历史输入中的 `custom_tool_call` / `custom_tool_call_output` 自动转换为 `function_call` / `function_call_output`，多轮 apply_patch 对话不再被 mimo 拒收。
- 隐藏商汤（SenseNova）生图模型 `sensenova-u1-fast`，避免在聊天模型列表里误选。
- 商汤聊天模型（`sensenova-*`）在 deepseek profile 下自动剥离 `reasoning_effort` / `thinking` / `parallel_tool_calls`，避免商汤上游 400。
- deepseek codex 账号端点切回 Chat Completions 协议，绕过 Responses 翻译层。
- mimo Responses 工具兼容性：只允许 `function` 类型；`custom` 类型自动转 `function`；`web_search` / `web_fetch` / `image_generation` / `file_search` / `code_interpreter` 等原生工具丢弃；不支持的 `tool_choice.type` 返回 409。
- Responses 直连下 `function_call.arguments` 规范化：空串 / object / 非法 JSON 一律修正为合法 JSON 字符串，避免 minimax 2013 错误。
- 退役 codex model mapping path：Codex 账号统一走 Responses 端点直选模型，model_map 仅作兼容保留。
- Codex 第三方账号自动路由到 Responses 协议。
- Chat 转换入口补齐工具历史，避免连续多轮工具调用上下文丢失。
- 适配 MiniMax 原始 tool_call 格式，原始 argument 不再被二次解析。
